How to Use

1. Connect to your SPD3303X

USB (Chrome/Edge only)
Click USB and select the virtual serial port. Uses the Web Serial API for direct hardware access — no extra software needed. Serial config: 9600 baud, 8N1.
Bridge (any browser)
Start the bridge: uv run bridge.py, then click Bridge. This relays SCPI commands over WebSocket so Firefox, Safari, and remote browsers can talk to the device.
Linux: device not showing up?
On Linux the SPD3303X presents as a USBTMC instrument at /dev/usbtmc0 rather than a serial port — the bridge detects and uses this automatically. To grant access without sudo, add a udev rule:
echo 'SUBSYSTEM=="usbmisc", KERNEL=="usbtmc*", ATTRS{idVendor}=="f4ec", MODE="0666"' | sudo tee /etc/udev/rules.d/99-siglent-spd3303x.rules
sudo udevadm control --reload-rules && sudo udevadm trigger

2. Toolbar buttons

Play / Stop
Start or stop automatic polling. Sends 4 measurement queries per cycle to read CH1/CH2 voltage and current. Status (CV/CC, output state) is polled every 2 seconds.
Record / Save
Record captures every reading with a timestamp. Save stops and downloads a CSV file with columns: Timestamp, CH1_Voltage_V, CH1_Current_A, CH2_Voltage_V, CH2_Current_A.

3. Controls

Expand the Controls section to set voltage/current for each channel and toggle outputs ON/OFF. Changes are sent immediately via SCPI commands.

4. Charts

Two stacked charts show voltage and current for both channels (CH1 red, CH2 blue). Window sets how much history to show. Clear erases all data and resets statistics.

5. Demo mode

Click Demo to generate fake power supply data (CH1: ~5V/0.5A, CH2: ~12V/1.0A) and try out the interface without hardware.

6. Typical workflow

Connect → set voltage/current → toggle output ON → Record → Play → monitor live readings → Stop → Save to download CSV.

Running locally

Don't open index.html directly — ES modules need a web server. Run uv run serve.py and open http://localhost:8000.

CommandDescription

CH1:CURRent 1.000Set CH1 current limit
OUTPut CH1,ONEnable CH1 output
OUTPut CH1,OFFDisable CH1 output

Status Register Bits

BitMeaning
0CH1 mode: 0=CV, 1=CC
1CH2 mode: 0=CV, 1=CC
2–3Tracking: 01=Independent, 11=Series, 10=Parallel
4CH1 output: 0=OFF, 1=ON
5CH2 output: 0=OFF, 1=ON

Serial Config

Virtual USB serial. 9600 baud, 8 data bits, 1 stop bit, no parity. Commands terminated with \n. Responses terminated with \n.
